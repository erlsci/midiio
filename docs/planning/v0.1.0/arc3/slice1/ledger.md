## Notes / disclosed deferrals

- **F1 is the carry-over headline.** Rows 3–6 + the tripwire (5) are the structural
  close of the arc2/slice2 UAF; the doc-only remediation (`F1-remediation-*`) states
  the contract, this slice makes it true. Don't ship without row 5 clean.
- **Mutex-vs-resource lifecycle (row 2)** is the subtle trap: the mutex is a field
  of the resource the destructor frees, so it must be destroyed *last*. Take it from
  the `nif-resources` card; this is a `NIF-LEARNINGS` candidate on close.
- **Cross-thread `enif_send` (rows 12–13)** is the segfault-risk path — process-
  independent env, NULL caller_env, SysEx copied during the callback. ASan owns it.
- **Sizing:** if this doesn't fit one context, split **1a** (lifecycle + lock + F1
  retrofit + keep/release + tripwire) / **1b** (recv term-build + seam + timestamp +
  loopback) — disclose at the cc-prompt.
- Out of scope: the conformance taxonomy + U1–U3/S1 (slice 2), UMP, WinMM, public
  virtual ports, batch, monitor-based owner-death cleanup.
